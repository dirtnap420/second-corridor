# Design & visual improvement plan — 50 items

Drafted 2026-06-10 from a full review of the rendered site (desktop 1280 / tablet 768 /
mobile 375, years 2022/2026/2030/2045) and the source. Screenshots used for the review
are in `qa/shots/sections/` (generated by `qa/sections.mjs` and `qa/instrument.mjs`,
both dev-only like `qa/screenshot.mjs`).

All items respect the locked design system (no border-radius, no gradients, no
backdrop-filter — enforced by `scripts/lint-design.mjs`). Items that brush against the
lint rule say so explicitly.

Priority key: **P1** = visible defect, fix first · **P2** = high-value polish ·
**P3** = enhancement.

---

## A. Rendering defects observed in screenshots (1–8)

1. **P1 — Fig 02 legend clips at the right plate edge.** At column width the key text
   `SUPPLY (STACKED ON PERM)` overflows the viewBox (`src/chart.js:117` — `keyX = W-332`
   is too tight for the rendered string). Fix by right-anchoring the key items or moving
   the key to a left-aligned row above the plot. (Superseded by item 23 if adopted.)

2. **P1 — Mobile site-panel caption collision.** At 375px the left caption
   `WHITE PINE SITE — SITEWORK 2026` overprints the right caption
   `DIAGRAMMATIC · SEQUENCE PER FEIS [12] & MILESTONES [11]` (see
   `qa/shots/sections/m-instrument.png`). Stack the two caption spans into block flow
   below ~560px (same flex-wrap treatment as `.plate-caption`). Source: `src/site.js`.

3. **P1 — Talent sankey is illegible at mobile widths.** At 375px node labels clip
   (`icroelectronic & electrical eng.`), the top program label is cut off by the
   viewBox, and mid-column labels sit on the ribbons (`m-s04.png`). Below ~560px render
   a stacked "route list" (school → program → employer rows with copper program chips)
   instead of the sankey, or give the SVG a min-width with horizontal scroll.

4. **P1 — Fig 02 milestone flags crowd on mobile.** The rotated year flags for
   2022–2030 overlap each other at 375px (`m-s02.png`). Drop flag *labels* below ~620px
   (keep hairlines), or label only first/last + milestone years with even spacing.
   Source: `src/chart.js:72-81` (no width gate today).

5. **P1 — Section-view riser labels collide.** Rotated value labels (`$20.0B`, `670`)
   overlap the node squares and each other at Clay/RIT (`i-section-view.png`). Place
   values horizontally above each bar; replace the rotated `QCEW SUPPR.` strings with a
   dagger (†) and one footnote line. Source: `src/map.js:176-214`.

6. **P1 — Albany riser clips at the right edge of the section view.** `SM.r = 30`
   (`src/map.js:62`) leaves no room for the last riser's labels; raise to ~56 in
   section view.

7. **P1 — Duplicated caption on Fig 04.** `STRUCTURE: PUBLIC RECORD · WIDTHS:
   ILLUSTRATIVE` appears in both the top and bottom plate captions
   (`index.html:200-207`). Keep the bottom one; restore the top-right slot to a
   fig-specific note or leave it empty.

8. **P1 — Node plate run-in spacing.** `dt`/`dd` are both `display:inline` with no
   gap, so the plate renders `NODECLAY — Micron` (`i-nodeplate.png`). Add
   `.node-plate dt { margin-right: 6px }` (`src/styles.css:474`).

## B. Citation system (9–12)

9. **P2 — Cites should land on their exact source row.** Every cite anchor points at
   `#sources`, though per-row ids `src-N` already exist (`main.js:443`). Point generated
   cites (`bindCiteMarks`, ledger marks `main.js:218`, dial marks `main.js:262`,
   installed-base `main.js:459`, node plate `main.js:469`) and the static `index.html`
   cites at `#src-N`.

10. **P2 — Highlight the jumped-to source.** Add
    `#sources li:target { background: var(--paper); border-left: 3px solid var(--copper); }`
    (with padding compensation) so a cite click visibly lands somewhere.

11. **P2 — Anchor offset.** `scroll-margin-top: 24px` on `section.sheet` and
    `.sources li` so anchor jumps don't kiss the viewport edge.

12. **P3 — Stop superscripts distorting line-height.** `.cite` uses
    `vertical-align: super`, which opens leading gaps in body copy. Use
    `position: relative; top: -0.4em; vertical-align: baseline`
    (`src/styles.css:219-226`).

## C. The corridor instrument (13–22)

13. **P2 — Auto-select Clay on boot.** The node plate's empty state
    (`SELECT A NODE FOR ITS DATA PLATE`) is the first thing most visitors see under the
    map. Call `showNodePlate` with the hero node after mount (`main.js` boot), keeping
    the empty state only as fallback.

14. **P2 — Node hover state.** `.node-marker` only changes on `.active` (year-driven).
    Add `:hover rect.mark { fill: var(--silicon); stroke-width: 1.5 }` so the five
    squares read as clickable.

15. **P2 — Visible "selected" state.** Selecting a node updates the data plate but
    leaves the map unchanged. Add a persistent selection ring (offset copper square
    outline) distinct from the year-driven active fill, cleared when another node is
    chosen.

16. **P2 — Paper halo on node labels.** Apply the `paint-order: stroke` trick already
    used by `.sankey-label` to `.node-marker text` so RIT/CLAY stay crisp where the
    copper trace crosses them.

17. **P3 — Map furniture in dead space.** A mono scale bar (0–50 mi) and small north
    arrow in the empty lower-left of the map SVG. Cheap, on-register for a "field
    instrument," and fills the dead corner left by NY's shape.

18. **P2 — Move the scrubber up under the map.** Copy says "Scrub the timeline," but
    the scrubber sits below site panel + node plate + dials. Reorder inside
    `#instrument`: map → scrubber → site panel → node plate → dials → ledger
    (`index.html:95-125`).

19. **P1 — Lock the Play button width.** `PLAY` → `PAUSE` → `REPLAY` relabeling
    resizes the grid column and reflows the track. `#play { min-width: 9ch }`.

20. **P3 — Milestone tick tooltips.** The copper scrubber ticks are unexplained. Give
    `.tick--milestone` `pointer-events: auto` and a `title` ("2026 — Groundbreaking;
    Fab 1 construction begins"), built in `buildTicks()` (`main.js:196`).

21. **P2 — Ledger affordances.** Rows glide the timeline on click but look inert. Add
    `cursor: pointer`, a hover background (`var(--paper)`), and an overflow hint (the
    list scrolls inside 280px with no cue — a bottom hairline + `▼ N MORE` mono chip;
    note a gradient fade-mask would trip the design lint).

22. **P3 — Tie the site panel to Clay.** The isometric fab panel appears under the
    state map with no linkage. Caption it `DETAIL — CLAY NODE · WHITE PINE SITE` and/or
    draw a hairline leader from the Clay marker, so the scale jump reads as deliberate.

## D. Figures 02–05 (23–29)

23. **P2 — Fig 02: direct end-labels instead of a legend.** Label each series at its
    right edge (`PERM 9,000`, `SUPPLY ~41,000`, hatch swatch `CONSTR BAND` near the
    band) — clearer than the key and permanently resolves item 1.

24. **P2 — Fig 02: hover-scrub the chart.** Pointer-tracking crosshair previewing
    values on hover; click commits the year to the master scrubber via the existing
    `glideTo`. Every chart becoming a controller is the site's signature move and the
    plumbing already exists.

25. **P2 — Fig 03: magnified inset for sub-$2B flows.** Micron's $100B band crushes
    Workforce ($65M) and Albany EUV ($825M) to the 1.25px floor the plate already
    apologizes for. Add a `×40 DETAIL` strip below the sankey re-drawing only the small
    flows at magnified scale, captioned as such — honesty preserved, legibility gained.

26. **P3 — Fig 03: four sources, four identities.** `chipsdirect` and `fedrd` both
    render violet (`src/sankey.js:87-92`). Give federal R&D a hatch fill or distinct
    stroke so the capital stack's four origins are visually separable.

27. **P3 — Fig 04: emphasize the program spine.** The copper mid-column is the
    figure's argument. Raise `nodeWidth` for mid nodes (12 → 16) and set program labels
    at weight 500 so routes read as passing *through programs*, not just left-to-right.

28. **P3 — Fig 05: resolve the orphan grid cell.** Seven cells in a 4-col grid leaves
    a hole. Add an eighth cited stat, or set the last cell as a typeset source line
    (`ALL FIGURES: STATE & INSTITUTIONAL RECORD`).

29. **P3 — Fig 05: bigger, steadier numerals.** `.spec-cell .big` 28px → 32–34px with
    `font-variant-numeric: tabular-nums`; align cite marks consistently against the cap
    height (they currently float at varied heights next to `CLASS 1000` / `150 MM`).

## E. Live figures 06–11 (30–36)

30. **P2 — Fig 06a: compress suppressed counties.** Four of seven county panels are
    identical full-size hatch boxes. Render suppressed counties at half height in a
    grouped "suppressed" sub-row so Monroe and Onondaga — the data — carry the plate.

31. **P3 — Fig 06a: explicit zero state.** Genesee (`no covered NAICS-3344
    employers`) draws an empty 78px plot. Draw a zero-baseline tick row so the empty
    space reads as a measured zero, not a rendering failure.

32. **P3 — Fig 06c: totals atop bars + width.** Five slim stacked bars float in
    whitespace at 1100px. Widen bars / tighten gaps and print each year's total above
    its bar.

33. **P2 — Fig 07: distinguish the self-commute bar.** Row 1 (Onondaga, 61.8%) is
    residents working in their own county — not catchment. Render it hatched/hollow as
    a reference row so the out-of-county story sets the visual scale. (Keeps the
    no-causal-claims discipline: same data, honest emphasis.)

34. **P3 — Fig 09: mark the data edge.** Each strip just stops. A small `TODAY · M44`
    tick at the last measured month distinguishes "data ends" from "suppressed" from
    "zero."

35. **P1 — Fig 10a: right-edge clipping.** The dotted `GB. '2x` groundbreaking marker
    label clips at every panel's right edge (visible in all six panels at 1280). Reserve
    right margin in the small-multiple layout or end-anchor the label inside the panel.

36. **P3 — Fig 11: one legend, grouped rules.** Replace the per-row `planned` labels
    with a single legend line (`INK MEASURED · GRAY APPLIED-FOR · COPPER PLANNED`) and
    add a hairline divider above the `WATER · MGD` group header.

## F. Typography & micro-layout (37–41)

37. **P2 — `text-wrap: balance` on `h1, h2`; `text-wrap: pretty` on `.body-copy`.**
    Kills widows like "then operations." for one line of CSS (progressive enhancement).

38. **P3 — Eyebrow rules.** Extend a hairline from each section eyebrow to the column's
    right edge (flex + `::after`), technical-drawing style — stronger section rhythm
    while scrolling.

39. **P3 — Brand selection color.** `::selection { background: var(--copper); color:
    var(--paper) }`.

40. **P3 — Instrument scrollbars.** `.ledger ol { scrollbar-width: thin;
    scrollbar-color: var(--muted) var(--paper) }` (plus numbers tables that overflow).

41. **P3 — Tabular numerals on data surfaces.** `font-variant-numeric: tabular-nums`
    on `.dial-value`, `.scrubber-year`, `details.numbers td`, `.spec-cell .big` —
    prevents jitter as values tick during play.

## G. Color & theme (42–44)

42. **P2 — Add a `--copper-text` token for accessibility.** Copper `#b5562a` on the
    silicon ground measures ≈4.1:1 — below WCAG AA (4.5:1) for the 12px eyebrows and
    masthead sub-line. Darken text-only usages to ≈`#9c4a22` (≈4.9:1 on silicon, 5.6:1
    on paper); keep `--copper` for marks, fills, and large type.

43. **P3 — Dark "night-plot" theme.** Tokens are already centralized in `:root`;
    define an inverted set (ground `#14161d`, paper→ink, copper unchanged) behind
    `prefers-color-scheme` + a manual toggle. Lint-safe. Prerequisite: the canvas
    particle layer hard-codes `'#b5562a'` (`src/map.js:403,422` and the frame loop) —
    read the token via `getComputedStyle` at draw time. Poster/OG export stay light.

44. **P3 — Registration marks on the main plate.** Corner crop-marks on `.plate--main`
    via `::before`/`::after` hairlines — plotter-sheet aesthetic, ~10 lines of CSS, no
    banned properties.

## H. Mobile (45–47)

45. **P2 — Compact mobile dials.** At <640px each dial takes a full row, pushing the
    scrubber ~3 screens below the map (`m-instrument.png`). Render dials as one compact
    3-across row (value + label only, no gauge SVG) on mobile, or a horizontal scroll
    row — the scrubber must stay near the map.

46. **P2 — Tighter mobile map crop.** `H = W*0.55` gives a 206px-tall full-state map
    at 375px — the corridor is tiny and the flanks are empty. Below ~600px, `fitExtent`
    to the corridor's bounding box (plus padding) instead of the whole state; keep the
    full state from 768px up.

47. **P3 — Order QCEW small multiples by signal on mobile.** The stacked order buries
    Monroe/Onondaga (the two counties with data) beneath Genesee's zero panel. Sort:
    data first, zero next, suppressed last, when stacked single-column.

## I. Access & interaction polish (48–50)

48. **P2 — Skip link.** A visually-hidden-until-focus "Skip to the corridor
    instrument" link before the masthead; the page is one long scroll with heavy
    pre-instrument chrome for keyboard users.

49. **P3 — Mini-TOC rail at ≥1400px.** A fixed right-edge mono index (`01`–`11` +
    `SRC`), current section highlighted via IntersectionObserver. Twelve sheets is a
    long scroll with no wayfinding; at wide viewports the rail costs nothing.

50. **P2 — Calm the year live region during play.** `output#year-readout` is
    `aria-live="polite"` and announces every year (~2/sec for ~13s) while playing. Set
    `aria-live="off"` while `state.playing`, then announce the final year once on
    pause/stop (`main.js` startPlay/stopPlay).

## Suggested sequencing

- **Pass 1 — defects (P1):** 1, 2, 3, 4, 5, 6, 7, 8, 19, 35 — everything a reader can
  see broken today.
- **Pass 2 — high-leverage polish (P2):** 9–11, 13–16, 18, 21, 23–25, 30, 33, 37, 42,
  45, 46, 48, 50.
- **Pass 3 — enhancements (P3):** the rest; 43 (dark theme) and 49 (TOC rail) are the
  largest single items and can ship independently.

After each pass: `npm run qa` + `node qa/sections.mjs` to regenerate review shots, and
`npm run build` to confirm the design lint stays clean.
